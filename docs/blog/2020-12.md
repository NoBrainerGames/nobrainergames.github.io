<sup>First 3D avatar in Nim.</sup>

This scene was the product of the addition of two systems: (1) the physics engine and (2) an entity component system. The first system established gravity in the scene, allowing the avatar to "fall" to the floor. The second system enabled navigation using the mouse.

The physics system wrapped AMD's [FEMFX](https://gpuopen.com/femfx/) library, which was a newly opensourced project at the time. I considered going with Nvidia's [PhysX](https://www.nvidia.com/en-us/drivers/physx/physx-9-19-0218-driver/) library initially, but after some research I concluded that I should go with a smaller library that I could contribute to.

The entity component system (ECS) implementation was inspired by skypjack's amazing [series on ECS](https://skypjack.github.io/2019-02-14-ecs-baf-part-1/). I decided to implement a pure Nim version instead of wrapping his library to better take advantage of Nim's language features.
